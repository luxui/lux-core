Type Definitions

Context

Application components can optionally recieve a context object, as an instance parameter, for interacting with or manipulating the application at the top-level in a consistent through the provided methods. The "context" feature of React is a way to provide this object implicitly to child components without the need to explicitly "passing down" the object through all of the layers of components.

Type:
  • Object
Properties:
Name Type Description
request function

perform a root-relative API request; where the root is prefilled by the value provided at application init.
setState function

pass the new state object to change the current state of the application; state change will incur a re-render.
state Object

the current state of the application; a copy of the actual state instead of a reference to the actual state object.
visit function

pass an application path to naviate to the corresponding page; an API request will be made if necessary.

Example
// define a new application component
function MessageBoard(props, context) {
  // ...
}

  setState: React.PropTypes.func,
};
